# Cube Split Timer 開発ルール

このプロジェクトは Cube Split Timer です。

詳細な開発計画は `docs/cube_split_timer_full_plan.txt` にあります。
作業を始める前に必ずそのファイルを読んでください。

## 共通方針

- React + Vite + TypeScript で実装する
- 不要なライブラリは入れない
- タイマー中は通信や重い処理をしない
- TypeScriptの型を丁寧に定義する
- 既存機能を壊さない
- 変更したファイルと実行コマンドを説明する
- エラーが出たら修正方針を説明してから直す
- service_role key は絶対にフロントエンドに置かない
- RLSが必要な機能では、安全なポリシーを前提にする

## 作業ルール

- ユーザーが指定したPhaseだけ実装する
- 指定されていないPhaseを勝手に実装しない
- 実装後、確認すべき動作チェック項目を出す
