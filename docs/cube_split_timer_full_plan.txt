--------------

「実装しやすい順番」に並べ替えてまとめたものです。

最重要ルール
------------
1. タイマー中は絶対に軽くする
- Supabase通信しない
- localStorageへ頻繁に保存しない
- 解析処理しない
- ランキングやフレンド情報を読み込まない
- 下ナビや設定ボタンを隠す

2. 保存はソルブ終了時だけ
- 計測中は表示だけ
- 停止したらlocalStorageに保存
- ログイン済みならその後Supabaseに同期

3. 削除はsoft delete
- 完全削除ではなく deletedAt / deleted_at を入れる
- 通常履歴には表示しない
- 平均・ao5・ao12・グラフ・best・worstから除外
- 管理者画面では削除済みも確認・復元可能

4. 重い機能は別ページ
- /timer は軽くする
- /analyzer は解析と3Dキューブ
- /learn は手順学習
- /ranking はランキング
- /friends はフレンド
- /admin は管理者

5. スマホファースト
- 大きいタイマー
- ボタンは押しやすく
- 計測中は誤タップ防止
- PWA対応
- 長押し文字選択を防ぐ
- スクロール誤動作を防ぐ

実装順番
========

チャットで話した順番ではなく、作りやすさと安全性を優先した順番。


Phase 1: React + Vite + TypeScriptで基本タイマー
----------------------------------------------
目的:
まずはログインなしで、普通に使えるタイマーを作る。

作る機能:
- 通常タイマー
- Spaceキーまたは画面タップで開始・停止
- ミリ秒表示
- 3x3スクランブル生成
- タイム履歴
- localStorage保存
- +2 / DNF
- soft delete用 deletedAt
- average / ao5 / ao12 / best / worst
- スマホファーストUI
- ダークモード風UI

この時点ではSupabaseなし。
まずは毎日使えるローカルタイマーを完成させる。

Codex指示文:
React + Vite + TypeScriptで、ルービックキューブ用Webタイマーを作ってください。
まずはログイン機能なしで、localStorage保存だけにしてください。

機能:
- 通常タイマー
- Spaceキーまたは画面タップで開始・停止
- ミリ秒表示
- 3x3スクランブル生成
- タイム履歴
- localStorage保存
- +2 / DNF
- 削除は完全削除ではなく soft delete 用に deletedAt を持たせる
- average / ao5 / ao12 / best / worst は deletedAt がない記録だけで計算する
- スマホファーストのUI
- ダークモード風のシンプルUI

軽量化:
- 不要なライブラリを使わない
- タイマー表示は requestAnimationFrame で更新する
- localStorage保存はソルブ終了時だけ
- 履歴表示は最新100件まで


Phase 2: UI改善・テーマ切替・誤操作防止
--------------------------------------
目的:
スマホでもPCでも使いやすくする。
計測中の誤タップ、スクロール、文字選択を防ぐ。

作る機能:
- ダーク / ライト / システム設定テーマ
- 計測中は秒数だけ表示する専用画面
- 計測中は下ナビ、設定、プロフィールボタンを非表示
- タイマー文字とスクランブル文字をコピー不可
- 長押しの青い文字選択を防ぐ
- 計測中スクロール禁止
- Spaceキーのページスクロール防止
- リロード・戻る対策
- currentSolveDraft

仕様:
- 通常時は普通にスクロール可能
- 計測中だけ body に timing クラス
- 計測途中でリロードされたら、そのソルブは正式保存しない
- 起動時に未完了draftがあれば破棄する

Codex指示文:
UIを見やすく使いやすく改善してください。

要件:
- ダークモード / ライトモード / システム設定に合わせる の3つのテーマ切替を実装する
- 初期テーマはダークモードにする
- メインのタイマー画面は情報を詰め込みすぎず、Scramble、大きなタイマー、最低限の ao5 / ao12 だけ表示する
- モバイルでは下部ナビゲーションを表示する

計測中のUI:
- タイマー計測中は誤タップ防止のため、下部ナビゲーション、設定ボタン、プロフィールボタンなどを非表示にする
- 計測中は専用のシンプル画面に切り替える
- 通常タイマーモードでは、計測中は秒数だけを大きく表示する
- 停止後は通常UIに戻る
- 計測中に他ページへ遷移できないようにする

誤操作防止:
- タイマー画面では文字選択できないようにする
- スクランブル文字も選択・コピーできないようにする
- CSSで user-select: none; -webkit-user-select: none; -webkit-touch-callout: none; を適用する
- PCでSpaceキーを押した時にページがスクロールしないように preventDefault() する
- 計測中は body に timing クラスを付ける
- body.timing では overflow: hidden; position: fixed; width: 100%; を使ってスクロールを防ぐ
- 計測中の timer-screen には touch-action: none; overscroll-behavior: none; を適用する
- 計測していない時は普通にスクロールできるようにする

リロード・ページ離脱対策:
- 計測中にリロードやページ離脱しようとしたら beforeunload で警告を出す
- ただしスマホ/PWAでは警告が確実ではないので、警告に依存しない設計にする
- ソルブ開始時に currentSolveDraft を localStorage または sessionStorage に保存する
- currentSolveDraft には status: "running", startTime, scramble, splits を保存する
- 正式な solves 履歴には、タイマー停止時だけ保存する
- 計測途中でリロードされた場合、次回起動時に running の currentSolveDraft を検出し、「前回の計測は未完了のため保存しませんでした」と表示する
- 未完了ソルブは正式履歴、平均、ao5、ao12、グラフに含めない


Phase 3: CFOP Split Mode
-----------------------
目的:
Cross / F2L / OLL / PLL の各工程タイムを測れるようにする。

操作:
- Sキーではなく、Space / Tap だけで進める
- Space 1回目: スタート
- Space 2回目: Cross完了
- Space 3回目: F2L完了
- Space 4回目: OLL完了
- Space 5回目: PLL完了＆停止

保存:
- mode = cfop_split
- totalTime
- crossTime
- f2lTime
- ollTime
- pllTime
- scramble
- penalty
- deletedAt
- createdAt

誤作動対策:
- KeyboardEvent.repeat が true の場合は無視
- 1入力で1段階だけ進む
- 200ms程度のdebounce
- preventDefault
- Spaceキーは keydown ではなく keyup で状態を進める
- キーを押しっぱなしにしている間は状態を進めない
- Spaceを離したタイミングで1回だけ処理する
- keydownでは preventDefault のみ行い、タイマー状態の変更はしない

Codex指示文:
CFOP Split Modeを追加してください。

仕様:
- Cross / F2L / OLL / PLL を計測する
- Sキーではなく Spaceキーだけで操作する
- Space 1回目でスタート
- Space 2回目でCross完了を記録
- Space 3回目でF2L完了を記録
- Space 4回目でOLL完了を記録
- Space 5回目でPLL完了＆タイマー停止
- スマホでは画面タップで同じ操作ができるようにする
- タイマー本体は途中で止めない
- totalTime, crossTime, f2lTime, ollTime, pllTime を保存する
- 各フェーズの平均、最速、最遅を表示する

誤作動対策:
- Spaceキーでページがスクロールしないように preventDefault() する
- KeyboardEvent.repeat が true の場合は無視する
- 1回の入力で1段階だけ進むようにする
- 200ms程度のdebounceを入れて二重反応を防ぐ
- Spaceキーは keydown ではなく keyup で状態を進める
- keydownでは preventDefault のみ行う
- キーを押しっぱなしにしている間は状態を進めない
- Spaceを離したタイミングで1回だけ処理する

表示:
- 計測中は秒数を大きく表示する
- CFOP Split中は現在フェーズ Cross / F2L / OLL / PLL を表示する
- すでに記録済みのスプリットは小さく表示する
- 停止後に Total, Cross, F2L, OLL, PLL を表示する


Phase 4: 保存しない・履歴削除・soft delete
-----------------------------------------
目的:
ミスソルブや測定ミスを扱えるようにする。

仕様:
- ソルブ終了後は基本自動保存
- 終了直後に「保存しない」ボタンを表示
- 保存しないを押すと deletedAt を入れる
- 履歴からも削除できる
- 削除もsoft delete
- 削除済みは平均・ao・グラフ・best・worstに含めない
- 削除直後に「元に戻す」

Codex指示文:
ミスソルブ対策として soft delete を追加・整理してください。

仕様:
- ソルブ終了時は基本的に自動保存する
- 終了直後に「保存しない」ボタンを数秒表示する
- 「保存しない」を押したら deletedAt に現在時刻を入れる
- 履歴の各記録にも「削除」ボタンを付ける
- 削除も完全削除ではなく deletedAt を入れる
- deletedAt がある記録は通常履歴に表示しない
- average / ao5 / ao12 / ao50 / ao100 / best / worst / グラフ / フェーズ平均には含めない
- 削除直後に「元に戻す」を表示する
- 復元する場合は deletedAt を null に戻す


Phase 5: Practice Mode
---------------------
目的:
通常ソルブ以外の練習をできるようにする。

作る練習:
1. Cross Practice
2. F2L Practice
3. F2L Pair Split

Cross Practice:
- Space / Tap でスタート
- 次の Space / Tap でクロス完了＆停止
- mode = cross_practice
- crossTime
- scramble
- crossColor
- deletedAt

F2L Practice:
- クロスを作った後、F2Lだけ測る
- Space / Tap でスタート
- 次の Space / Tap でF2L完了＆停止
- mode = f2l_practice
- f2lTime

F2L Pair Split:
- F2Lの4ペアをそれぞれ測る
- Space / Tap でスタート
- Pair1 / Pair2 / Pair3 / Pair4 をSpaceで記録
- Pair4で停止
- mode = f2l_pair_split

Codex指示文:
Practiceページに Cross Practice と F2L Practice と F2L Pair Split を追加してください。

Cross Practice:
- mode = "cross_practice"
- Space / Tap でスタート
- 次の Space / Tap でクロス完了＆停止
- crossTime を保存
- scramble, crossColor, createdAt, deletedAt を保存
- Cross Practice専用の履歴、平均、best、ao5、ao12を表示する

F2L Practice:
- mode = "f2l_practice"
- Space / Tap でスタート
- 次の Space / Tap でF2L完了＆停止
- f2lTime を保存
- F2L Practice専用の履歴、平均、best、ao5、ao12を表示する

F2L Pair Split:
- mode = "f2l_pair_split"
- Space / Tap でスタート
- その後の Space / Tap で Pair 1, Pair 2, Pair 3, Pair 4 を記録する
- Pair 4で停止する
- pair1Time, pair2Time, pair3Time, pair4Time, totalTime を保存する
- 各ペアの平均、最速、最遅を表示する

共通:
- ソルブ終了後は基本自動保存
- ミスった時だけ「保存しない」
- 履歴から削除できる
- 削除はsoft deleteで deletedAt を使う
- 削除済み記録は平均、ao5、ao12、グラフに含めない

- 192x192 と 512x512 のアイコンを設定する

- オフラインでもlocalStorageに保存済みの記録は見られるようにする
- オフライン中に計測した記録はlocalStorageに保存し、オンライン復帰後にSupabaseへ同期できる構成にする
